It is a primary design goal for Aspen to elegantly support both threaded and cooperatively scheduled concurrency models. Right now everything is done except the “elegantly” part. You have:

  • the diesel event loop (wraps greenlet)
  • a diesel.thread(func, *a, **kw) call for one-off threads
  • non-blocking protocol implementations of varying quality bundled with diesel

See this design doc for details on where things are headed in terms of wrapping these in Aspen.
