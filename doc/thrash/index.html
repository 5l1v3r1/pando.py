Aspen comes with a utility called thrash that restarts a process whenever it dies. It is installed along with the aspen executable, via setuptools.

You can configure aspen to die whenever module files or files under a .aspen hidden directory in your website root change. That hidden directory is where you put template bases and includes, etc. To set this up, create an aspen.conf file with this content:

[aspen]
changes_kill = yes

This is how Aspen supports the common case of restarting a server in development when files change. In production you should use something like daemontools or supervisord. Note that simplate files are automatically reloaded without a server restart whenever they change, both in development and production.
