The ^L character is an ASCII page break, also called:

  • form feed
  • \f
  • 12
  • 0xC

Aspen uses the ASCII page break to split simplates into pages.

How to Type a Page Break

If you’re not comfortable working with page breaks, then Aspen is not for you. That said, here are some notes on various editors:

Vim Ctrl-L (in insert mode)
Emacs C-q C-l
UltraEdit There is a "Page Break" option on the "Insert" menu. You should remap Ctrl-L to that.
TextMate, etc. ???

Please email me if you have any additions to that list or if you need help figuring it out for your editor.

Rationale

Yes, Aspen uses a control character that may be new to you. This is a design tradeoff. The page break is a perfect fit semantically (a simplate is one file with several “pages”), and it is easy to type. However, the current tool support is weak in the long tail.

Tool Support

Emacs has good support for page breaks. It actually provides for navigating through a file based on the page breaks (this is how I learned of them, actually: Barry Warsaw uses them in the email package in the standard library). UltraEdit has the nicest presentation of them I’ve seen, but you have to configure your key mappings to use Ctrl-L for it. Page breaks work fine in Vim, though there’s no special treatment. Other editors may be less friendly. Notepad is pretty hopeless.

Ideally, one would be able to configure an editor to use different syntax highlighting for different pages in a file. For Aspen simplates, one would want the first two (optional) pages in a file to be Python, and the third to vary by file extension.

Web browsers don’t render page breaks, so any time you want to put a simplate code snippet on a website you have two options:

  • Use an actual page break, which will be confusingly invisible on the website, but will work if copied and pasted
  • Change the page break to "^" "L", which will show in the browser, but will confusingly break when copied and pasted into an editor

Alternatives

I did have a version of simplates at one point that used a goofy comment string to delimit pages, something like:

#<!-- BREAK -->

This is maybe easier for newbies to understand, but it’s ugly and harder to type. You could make it easier to type with a macro, but then you’ve made it harder for newbies again, and it’s still ugly. The page break is so cute:

If you like simplates but dislike the page break, then you have until Aspen 1.0 to suggest something that works perfectly with existing tools and isn’t ugly.
