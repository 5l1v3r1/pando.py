Aspen’s hooks.conf file uses colon notation to specify Python objects.

mypackage.sub.another:blah

becomes

from mypackage.sub import another

The blah attribute of another would then be used.

API

To use colon notation in your own application, import the colonize function from aspen.colon. Here is it’s signature:

def colonize(name, filename='n/a', lineno=-1):
    """Given a name in colon notation and some error helpers, return an object.
    """

And here’s a contrived example of how to use it:

from aspen.configuration import colon

objdef = "aspen.configuration.colon:colonize"
colonize = colon.colonize(objdef)
