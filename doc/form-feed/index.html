Rationale

Yes, you have to learn a new control character to use Aspen and simplates. This is a design tradeoff.

The points in favor of the form feed are that it fits semantically (a simplate is one file with several “pages”), it is easy to type, and it is little-used otherwise. The one other use of it that I know of is in Emacs, where you can page through a file based on the form feeds. This actually harmonizes well with Aspen’s use of the character.

The problems with using form feed are that it is unfamiliar to people (“I need a tutorial to type this character?”) and that it doesn’t show up in a web browser, so any time you want to put a simplate code snippet on a website you have two options:

  • Use an actual form feed, which will be confusingly invisible on the website
  • Change the form feed to "^" "L", which will confusingly break when copied and pasted into an editor

A third problem is that editors don’t support automatically switching syntax highlighting for different pages within a file, so you have to do that manually.

I take the second and third objections more seriously than the first. Reviving a control character that has fallen on hard times is quirky and fun, but adding undue friction to tutorials and daily editing is lame.

Regarding the second objection, I actually tried using a goofy comment string at one point, something like:

#<!-- BREAK -->

This answers the copy/paste objection, but is harder to type directly. You could configure a macro for this, but to me that’s more of a burden to new users than explaining how to type ctrl-L.

The bottom line, however, is that I haven’t found a character that has the same soul as the form feed. Look at him! He’s so cute:

Maybe some day browsers will display the form feed in a way that is sensible. Maybe some day text editors will support highlighting different pages of a file differently. Maybe some day the lowly form feed will get the respect it deserves—or maybe some day someone will suggest an alternative that is obviously better. But until then, form feed has a place in Aspen simplates.

Working with Form Feed

The ^L character is an ASCII form feed, also called:

  • page break
  • \f
  • 12
  • 0xC

Aspen uses ^L to split simplates and the hooks.conf file into sections. If memory serves, I first saw it in Barry Warsaw's email package in the standard library. Barry uses Emacs, which supports paging through a file based on the form feeds in it.

Here is how to type ^L in various editors:

Vim Ctrl-L (in insert mode)
Emacs C-q C-l
UltraEdit If I remember right, there is a "Page Break" option on the "Insert" menu. You’d probably want to remap that in the long term.
TextMate, etc. ???

Please email me if you have any additions to that list or if you need help figuring it out for your editor.
