Aspen can be configured via an aspen.conf file. This is an INI-style file per the standard library's RawConfigParser object. It is read from the following three locations. Later files override earlier.

  • /etc/aspen/aspen.conf
  • /usr/local/etc/aspen/aspen.conf
  • ~/aspen/aspen.conf
  • ./.aspen/aspen.conf

Aspen itself uses the following keys from the [aspen] section:

changes_kill

If set to yes, Aspen will exit when Python module files or any files under a .aspen hidden directory in the website root change. Use this option with thrash to bounce aspen in development.

Valid values: yes/no/true/false/0/1
Default: no
list_directories

If set to yes, Aspen will show a directory listing when there is no index.html in a given directory. If set to no, requests for such directories will result in a 404.

Valid values: yes/no/true/false/0/1
Default: no
show_tracebacks

If set to yes, Aspen will display debugging information for error responses. If set to no, a simple HTTP message will be shown without additional detail.

Valid values: yes/no/true/false/0/1
Default: no
