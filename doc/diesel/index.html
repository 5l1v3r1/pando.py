Aspen uses the Diesel library, version 2.0.0, for asynchronous network I/O. Here is where Diesel fits in the Python async landscape:

  • Diesel is a coroutine-based framework, unlike Twisted and Tornado, which use callbacks.
  • Diesel doesn't monkey-patch the standard library, unlike gevent.

Diesel has a particular emphasis on bundling its own non-blocking protocol implementations. It also includes support for farming out to one-off threads.
