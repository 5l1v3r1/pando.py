The template page of a simplate file uses the templating language from the Tornado framework (under the Apache license). It’s a Django-like syntax, but with arbitrary Python expressions allowed, and only a single {% end %} tag instead of separate tags for {% endif %}, etc.

Expressions

You can evaluate Python expressions inline like so:

{{ expression }}

Any objects imported or defined in the first two Python pages of the simplate are available here. This includes the request and response objects. Additionally, the following four helper functions are available:

def escape(value):
    """This is an alias for xhtml_escape."""

def xhtml_escape(value):
    """Escapes a string so it is valid within XML or XHTML."""

def json_encode(value):
    """JSON-encodes the given Python object."""

def squeeze(value):
    """Replace all sequences of whitespace chars with a single space."""

def url_escape(value):
    """Returns a valid URL-encoded version of the given value."""

Operators

Here are the operators you get:

For

{% for foo in bar %}
    ...
{% end %}

Extends/Block

Put base templates in the .aspen hidden directory in your website root. You can use subdirectories (see include for an example).

{% extends "foo.html" %}
{% block bar %}
    ...
{% end %}

If/Elif/Else

{% if foo %}
    ...
{% elif %}
    ...
{% else %}
    ...
{% end %}

Include

Put include files in the .aspen hidden directory in your website root. As shown here, you can use subdirectories of your choosing. Note though that directory separators are not automatically translated across operating systems, so the following would break on Windows.

{% include "bar/foo.html" %}

Try/Except/Finally

{% try %}
    ...
{% except %}
    ...
{% finally %}
    ...
{% end %}

While

{% while foo %}
    ...
{% end %}
{% end %}
